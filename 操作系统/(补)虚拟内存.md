# 虚拟内存  
```
CPU
 ↓
虚拟地址=虚拟页号+偏移       虚拟内存(虚拟页0 虚拟页1 虚拟页2 虚拟页3...)
 ↓
MMU+页表(虚拟页号->物理页号)
 ↓得到
物理地址 = 物理页号+偏移
```
```
虚拟页 -> 物理页      虚拟页 0x1000  →  物理页 0xA300
                      虚拟页 0x2000  →  物理页 0xB000

虚拟页 -> 文件(mmap)   虚拟页 0x1000 → data.bin offset 0
                      虚拟页 0x2000 → data.bin offset 4096

虚拟页 -> 磁盘（swap） 虚拟页 0x1000 → swap offset 123
                      虚拟页 0x1000 → swap offset 123

swap 是操作系统提前准备好的一块 专门用于存放被换出的内存页的磁盘区域
```
```
详细流程:
假设程序访问一个地址 虚拟地址0x2000
CPU会把地址交给 MMU,MMU查页表得到 虚拟页0x2000 →(present=0) 表示:页不在内存,触发page fault(缺页异常)
可能有三种情况
1.虚拟页0x2000 → fileA offset 8192(present=0) mmap文件页 
2.虚拟页0x2000 → swap offset 500(present) 数据在swap 
3.虚拟页0x2000 -> 匿名页 重新分配
找一个空闲物理页将数据放入
如果没有空闲物理页 挑选一个“冷页”,放入swap,腾出空间
最后更新页表
```
